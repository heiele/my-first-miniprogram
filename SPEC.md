# 学习日历小程序 - 需求规格说明书（SPEC）

## 1. 项目概述

### 1.1 项目背景
学习日历小程序是一款帮助用户管理学习任务、记录专注时间、养成良好学习习惯的微信小程序。通过整合任务管理、日历视图、专注计时、习惯打卡等功能，帮助用户更高效地规划学习计划，追踪学习进度。

### 1.2 目标用户
- 学生群体：需要管理学习任务、备考计划的中小学生和大学生
- 自学者：希望养成学习习惯、记录学习进度的自主学习者
- 职场人士：需要持续学习提升技能的职场人员

### 1.3 核心价值
- 高效任务管理：支持任务创建、优先级排序、完成状态追踪
- 专注计时：番茄钟计时，记录专注时段
- 习惯养成：打卡记录，连续打卡统计
- 数据统计：学习进度可视化，效率分析

## 2. 功能需求

### 2.1 任务管理模块（P0）

#### 2.1.1 查看任务
- **描述**：用户可以查看所有任务列表，包括今日任务和历史任务
- **优先级**：P0
- **需求点**：
  - 支持按日期筛选任务
  - 支持按优先级排序（高/中/低）
  - 支持智能排序（综合截止时间、优先级、兴趣度）
  - 显示任务完成状态

#### 2.1.2 添加任务
- **描述**：用户可以创建新的学习任务
- **优先级**：P0
- **需求点**：
  - 任务标题（必填）
  - 任务描述（选填）
  - 优先级设置（高/中/低，默认中）
  - 截止日期和时间（必填）
  - 兴趣度设置（0-100，默认50）
  - 支持添加子任务

#### 2.1.3 切换完成状态
- **描述**：用户可以标记任务为已完成或取消完成
- **优先级**：P0
- **需求点**：
  - 点击任务完成按钮切换状态
  - 已完成任务自动移至回收站
  - 支持从回收站恢复任务

#### 2.1.4 删除任务
- **描述**：用户可以删除不需要的任务
- **优先级**：P0
- **需求点**：
  - 删除前确认提示
  - 删除后任务移至回收站
  - 回收站任务可永久删除

### 2.2 日历视图模块（P1）

#### 2.2.1 月历展示
- **描述**：以月历形式展示学习计划
- **优先级**：P1
- **需求点**：
  - 当前月份日历视图
  - 标记有任务的日期
  - 点击日期查看当日任务

#### 2.2.2 打卡记录展示
- **描述**：日历上标记学习打卡情况
- **优先级**：P1
- **需求点**：
  - 已打卡日期高亮显示
  - 连续打卡天数统计

### 2.3 专注计时模块（P1）

#### 2.3.1 番茄钟计时
- **描述**：标准番茄钟计时功能
- **优先级**：P1
- **需求点**：
  - 默认25分钟专注+5分钟休息
  - 支持自定义时长
  - 计时状态展示（进行中/暂停/结束）

#### 2.3.2 专注统计
- **描述**：统计专注时间数据
- **优先级**：P1
- **需求点**：
  - 今日专注时长统计
  - 本周专注数据图表
  - 历史专注记录查看

### 2.4 习惯打卡模块（P1）

#### 2.4.1 习惯管理
- **描述**：创建和管理学习习惯
- **优先级**：P1
- **需求点**：
  - 创建习惯（名称、图标、目标天数）
  - 删除习惯
  - 查看习惯列表

#### 2.4.2 打卡功能
- **描述**：每日打卡记录
- **优先级**：P1
- **需求点**：
  - 每日一次打卡
  - 连续打卡天数统计
  - 打卡历史记录

### 2.5 个人中心模块（P2）

#### 2.5.1 用户信息
- **描述**：管理用户基本信息
- **优先级**：P2
- **需求点**：
  - 昵称、签名设置
  - 主题切换（浅色/深色）

#### 2.5.2 学习目标
- **描述**：设置和追踪学习目标
- **优先级**：P2
- **需求点**：
  - 每日学习时长目标
  - 学习类型选择
  - 目标截止日期

#### 2.5.3 数据统计
- **描述**：学习数据可视化
- **优先级**：P2
- **需求点**：
  - 任务完成率统计
  - 学习时长统计
  - 成就徽章展示

## 3. 非功能需求

### 3.1 性能要求
- 页面加载时间 ≤ 2秒
- 接口响应时间 ≤ 500毫秒
- 支持离线缓存，网络恢复后自动同步

### 3.2 可用性要求
- 网络断开时有友好的错误提示
- 操作失败时有重试机制
- 数据本地备份，防止数据丢失

### 3.3 兼容性要求
- 支持微信小程序基础库 2.20.0+
- 适配 iOS 和 Android 设备
- 适配不同屏幕尺寸

## 6. 页面结构

```
├── pages/
│   ├── index/          # 首页 - 学习概览、今日任务
│   ├── calendar/       # 日历页 - 月历视图、打卡记录
│   ├── task/           # 任务页 - 任务列表、添加/编辑任务
│   │   └── recycle/    # 回收站页
│   ├── focus/          # 专注页 - 番茄钟、专注统计
│   ├── mine/           # 我的页 - 用户信息、学习目标、数据统计
│   ├── auth/           # 授权页 - 微信登录授权
│   └── logs/           # 日志页 - 调试日志
```

## 7. 技术栈

### 7.1 前端

- WXML + WXSS + JavaScript

### 7.2 后端

- JSON 文件存储（开发阶段）

## 8. 开发环境

### 8.1 前置要求
- 微信开发者工具

### 8.2 启动方式
```bash

cd backend
npm install
npm start

# 小程序
使用微信开发者工具打开项目根目录
```

## 9. 部署说明

### 9.1 小程序发布
1. 在微信开发者工具中点击"上传"
2. 在微信公众平台提交审核
3. 审核通过后发布

### 9.2 后端部署
- 生产环境建议使用 SQLite 或 MySQL
- 配置服务器域名白名单
- 使用 HTTPS 协议
